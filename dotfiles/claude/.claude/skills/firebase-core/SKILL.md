# firebase_core

Mandatory foundation for all FlutterFire plugins. Must be initialized before any
other Firebase service is used.

## Setup

### 1. Install the FlutterFire CLI

```bash
npm install -g firebase-tools
dart pub global activate flutterfire_cli
firebase login
```

### 2. Generate `firebase_options.dart`

Run this from your Flutter project root. Re-run whenever you add a new platform
or change Firebase projects.

```bash
flutterfire configure
```

This command:

- Prompts you to select a Firebase project and target platforms (iOS, Android,
  Web, macOS)
- Creates or updates `lib/firebase_options.dart`
- Adds required Gradle plugins for Android (Crashlytics, Performance Monitoring)

**CLI options for CI / flavors:**

```bash
flutterfire configure \
  --project=my-firebase-project \
  --out=lib/firebase_options_prod.dart \
  --ios-bundle-id=com.example.app \
  --android-package-name=com.example.app \
  --account=ci@example.com \
  --no-apply-gradle-plugin
```

### 3. Add the package

```yaml
dependencies:
  firebase_core: ^3.0.0
```

## Initialization

Always initialize Firebase **before** calling `runApp`. Call
`WidgetsFlutterBinding.ensureInitialized()` first so platform channels are
ready.

```dart
import 'package:firebase_core/firebase_core.dart';
import 'firebase_options.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();

  await Firebase.initializeApp(
    options: DefaultFirebaseOptions.currentPlatform,
  );

  runApp(const MyApp());
}
```

`DefaultFirebaseOptions.currentPlatform` is generated by the FlutterFire CLI and
automatically selects the correct `FirebaseOptions` based on the current
platform at runtime.

### Initialization in GetX / service bootstrap

```dart
class FirebaseService extends GetxService {
  static FirebaseService get to => Get.find();

  Future<FirebaseService> init() async {
    await Firebase.initializeApp(
      options: DefaultFirebaseOptions.currentPlatform,
    );
    return this;
  }
}

// In bootstrap / main:
await Get.putAsync(() => FirebaseService().init());
```

## FirebaseOptions

`FirebaseOptions` is an immutable configuration object. All four fields below
are **required**. The rest are optional and service-specific.

```dart
const FirebaseOptions(
  // Required
  apiKey:            'AIza...',        // authenticates requests to Google APIs
  appId:             '1:123:android:abc', // uniquely identifies this app instance
  messagingSenderId: '123456789',      // Cloud Messaging sender ID
  projectId:         'my-project-id', // Firebase project ID

  // Optional — add as needed per service
  authDomain:        'my-project.firebaseapp.com',   // OAuth redirect (web)
  databaseURL:       'https://my-project.firebaseio.com', // Realtime Database
  storageBucket:     'my-project.appspot.com',       // Cloud Storage
  measurementId:     'G-XXXXXXXX',                   // Analytics (web)
  iosClientId:       '...',                           // Google Sign-In (iOS)
  androidClientId:   '...',                           // Google Sign-In (Android)
  iosBundleId:       'com.example.app',              // iOS bundle ID
)
```

### Accessing options at runtime

```dart
final app = Firebase.app();
final options = app.options;
print(options.projectId);
print(options.appId);
```

## Firebase Class API

`Firebase` is a static-only class (cannot be instantiated).

```dart
// Initialize the default app
Future<FirebaseApp> Firebase.initializeApp({
  String? name,           // omit for the default app
  FirebaseOptions? options,
})

// Retrieve an app by name (throws if not found)
FirebaseApp Firebase.app([String name = defaultFirebaseAppName])

// All currently initialized apps
List<FirebaseApp> Firebase.apps
```

**`defaultFirebaseAppName`** — the constant name `'[DEFAULT]'` used when no name
is supplied.

## FirebaseApp API

```dart
// App identifier (defaultFirebaseAppName for the default app)
String app.name

// The FirebaseOptions used during initialization
FirebaseOptions app.options

// Toggle automatic data collection (e.g. for GDPR compliance)
await app.setAutomaticDataCollectionEnabled(false);
bool app.isAutomaticDataCollectionEnabled

// Release resources (cannot delete the default app)
await app.delete();
```

Equality is based on `name` + `options`, so two `FirebaseApp` references to the
same app are equal.

## Multiple Firebase Projects

Use named app instances to connect to more than one Firebase project (e.g.
separate dev and prod projects, or connecting to a partner's project).

### Initialize a secondary app

```dart
await Firebase.initializeApp(
  name: 'secondary',
  options: const FirebaseOptions(
    apiKey:            'AIza...',
    appId:             '1:456:android:xyz',
    messagingSenderId: '456789',
    projectId:         'secondary-project',
    storageBucket:     'secondary-project.appspot.com',
  ),
);
```

### Use a secondary app with a FlutterFire plugin

Most plugins expose an `instanceFor(app:)` factory:

```dart
final secondaryApp = Firebase.app('secondary');

// Firestore
final firestore = FirebaseFirestore.instanceFor(app: secondaryApp);

// Auth
final auth = FirebaseAuth.instanceFor(app: secondaryApp);

// Storage
final storage = FirebaseStorage.instanceFor(app: secondaryApp);
```

### Multi-environment pattern (dev / prod)

```dart
// lib/firebase_options_dev.dart  — generated with --out flag
// lib/firebase_options_prod.dart — generated with --out flag

Future<void> bootstrapFirebase(AppFlavor flavor) async {
  final options = flavor == AppFlavor.prod
      ? ProdFirebaseOptions.currentPlatform
      : DevFirebaseOptions.currentPlatform;

  await Firebase.initializeApp(options: options);
}
```

> **Analytics caveat:** Analytics events are only logged for the **default**
> app. Secondary app analytics are silently ignored.

### List and inspect all apps

```dart
for (final app in Firebase.apps) {
  debugPrint('${app.name} → ${app.options.projectId}');
}
```

## Platform Notes

### Android

- `google-services.json` is placed in `android/app/`.
- For flavors, place per-flavor files at
  `android/app/src/<flavor>/google-services.json`.
- The Google Services Gradle plugin converts the JSON into Android string
  resources automatically.
- `Firebase.initializeApp()` with **no options** reads from these string
  resources (Android only).

```dart
// Android-only: read from google-services.json resources
if (defaultTargetPlatform == TargetPlatform.android && !kIsWeb) {
  await Firebase.initializeApp();
} else {
  await Firebase.initializeApp(options: DefaultFirebaseOptions.currentPlatform);
}
```

### iOS / macOS

- `GoogleService-Info.plist` is embedded in the Xcode project.
- For multiple targets/schemes, create separate plist files and assign them via
  **Target Membership** in Xcode.
- Do not bundle `GoogleService-Info.plist` if providing options at runtime — the
  two methods conflict.

### Web

- No native config file. All configuration comes from `FirebaseOptions` passed
  to `initializeApp`.
- `authDomain` is required for OAuth redirects.
- `measurementId` is required for Analytics.

### macOS entitlements

Add network entitlements to both `DebugProfile.entitlements` and
`Release.entitlements`:

```xml
<key>com.apple.security.network.client</key>
<true/>
```

## Anti-Patterns

| Anti-Pattern                                               | Why                                                                                                                       | Fix                                                                         |
| ---------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------- |
| Calling any Firebase plugin before `initializeApp`         | Throws `FirebaseException: No Firebase App '[DEFAULT]' has been created`                                                  | Always `await Firebase.initializeApp(...)` before accessing any service     |
| Missing `WidgetsFlutterBinding.ensureInitialized()`        | Platform channels not ready; `initializeApp` throws                                                                       | Call it as the very first line of `main()` before any `await`               |
| Calling `initializeApp` multiple times for the default app | Throws `duplicate-app` exception on the second call                                                                       | Guard with `Firebase.apps.isEmpty` or initialize once at startup            |
| Hardcoding `FirebaseOptions` values                        | Values differ per platform; easy to mix up                                                                                | Always use `DefaultFirebaseOptions.currentPlatform` from the generated file |
| Deleting the default app                                   | Throws `FirebaseException`                                                                                                | Only `delete()` named secondary apps                                        |
| Logging Analytics on a secondary app                       | Events are silently dropped                                                                                               | Analytics must use the default app                                          |
| Committing `firebase_options.dart` with secret API keys    | API keys are not secret for Firebase (they are restricted by platform) but the file should be regenerated per environment | Acceptable to commit; use Firebase Security Rules to protect data           |

## Examples

### Standard `main.dart`

```dart
import 'package:firebase_core/firebase_core.dart';
import 'package:flutter/material.dart';
import 'firebase_options.dart';

Future<void> main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Firebase.initializeApp(
    options: DefaultFirebaseOptions.currentPlatform,
  );
  runApp(const MyApp());
}
```

### Safe re-initialization guard

```dart
Future<void> initFirebase() async {
  if (Firebase.apps.isNotEmpty) return;
  await Firebase.initializeApp(
    options: DefaultFirebaseOptions.currentPlatform,
  );
}
```

### Secondary app for a dev/prod split

```dart
await Firebase.initializeApp(
  options: DefaultFirebaseOptions.currentPlatform, // default = prod
);

await Firebase.initializeApp(
  name: 'dev',
  options: DevFirebaseOptions.currentPlatform,
);

// Use dev Firestore in debug builds only
final firestore = kDebugMode
    ? FirebaseFirestore.instanceFor(app: Firebase.app('dev'))
    : FirebaseFirestore.instance;
```

### GDPR / consent-based data collection

```dart
final app = Firebase.app();

// Disable collection until user consents
await app.setAutomaticDataCollectionEnabled(false);

// Re-enable after consent
Future<void> onUserConsent() async {
  await app.setAutomaticDataCollectionEnabled(true);
}
```

### Inspect all initialized apps

```dart
void logFirebaseApps() {
  for (final app in Firebase.apps) {
    debugPrint('[Firebase] ${app.name} — project: ${app.options.projectId}');
  }
}
```

### Generated `firebase_options.dart` structure (reference)

```dart
// Auto-generated by FlutterFire CLI — do not edit manually.
// Re-generate with: flutterfire configure

import 'package:firebase_core/firebase_core.dart' show FirebaseOptions;
import 'package:flutter/foundation.dart'
    show defaultTargetPlatform, kIsWeb, TargetPlatform;

class DefaultFirebaseOptions {
  static FirebaseOptions get currentPlatform {
    if (kIsWeb) return web;
    switch (defaultTargetPlatform) {
      case TargetPlatform.android: return android;
      case TargetPlatform.iOS:     return ios;
      case TargetPlatform.macOS:   return macos;
      default:
        throw UnsupportedError('DefaultFirebaseOptions not configured for this platform.');
    }
  }

  static const FirebaseOptions android = FirebaseOptions(
    apiKey: 'AIza...',
    appId: '1:123:android:abc',
    messagingSenderId: '123',
    projectId: 'my-project',
    storageBucket: 'my-project.appspot.com',
  );

  static const FirebaseOptions ios = FirebaseOptions(
    apiKey: 'AIza...',
    appId: '1:123:ios:def',
    messagingSenderId: '123',
    projectId: 'my-project',
    storageBucket: 'my-project.appspot.com',
    iosClientId: '123-abc.apps.googleusercontent.com',
    iosBundleId: 'com.example.app',
  );

  // web, macos follow the same pattern
}
```
