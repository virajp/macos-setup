# Documentation Templates

This directory contains standardized templates and guidelines for documenting shell scripts, Fish scripts, and Justfiles.

## Files in this directory:

- **`Documentation-Guide.md`** - Complete documentation standards guide
- **`file-header-template.txt`** - Copy-paste file header template
- **`function-doc-template.txt`** - Copy-paste function documentation template
- **`zsh-script-template.zsh`** - Complete Zsh script template with examples
- **`fish-script-template.fish`** - Complete Fish script template with examples
- **`justfile-template.just`** - Complete Justfile template with examples

## Quick Usage:

1. **For new scripts**: Copy the appropriate complete template file and fill in the placeholders
2. **For existing scripts**: Copy-paste the header or function templates as needed
3. **For reference**: Consult the Documentation-Guide.md for detailed standards

## Template Placeholders:

Replace these placeholders when using the templates:
- `[Brief description of what this script does]`
- `[Your Name]` and `[email@domain.com]`
- `[YYYY-MM-DD when created]`
- `[Required tools, permissions, or setup]`
- `[How to run this script with examples]`
- `[External tools, packages, or files needed]`
- `[License type, e.g., MIT, Apache 2.0, etc.]`
- `[function_name]`
- `[What this function does]`
- `[Description of arguments/variables/conditions]`

## Consistency Benefits:

Using these templates ensures:
- ✅ Consistent documentation format across all scripts
- ✅ Easy onboarding for new team members
- ✅ Clear usage examples and error handling documentation
- ✅ Proper attribution and licensing information
- ✅ Standardized comment formatting
